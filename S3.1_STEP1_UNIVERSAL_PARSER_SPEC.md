## CRITICAL REQUIREMENT: 12-POSITION CONSUMPTION ARRAY

### Why This Matters

Standard FV sizing algorithms expect exactly 12 months of consumption data for:
- Accurate monthly averaging
- Seasonality analysis
- Equipment sizing
- Capacity factor calculations
- ROI projections

**Dynamic array sizing** causes:
- Index out-of-bounds errors
- Regression instability
- Conditional branching in deterministic calculations
- Hash drift on baseline tests

### Implementation

```javascript
// ALWAYS 12 positions
consumo_12_meses: [
  120, // 12 months ago
  130, // 11 months ago
  ...
  145  // 1 month ago (most recent)
]

// Length = 12 (MANDATORY)
// Never < 12, never > 12
// If data missing: impute using available month average
// Flag: dados_estimados = true if imputation occurred
```

### Imputation Algorithm

```
1. Extract all available consumption data from bill
2. Calculate average of available months
3. For each missing month position:
   - Replace 0 with calculated average
4. Result: 12-position array, deterministic
5. Set dados_estimados = true if any imputation
```

### Example

```
Bill contains: [120, 130, 140, 145, 150] (5 months only)
Average: (120+130+140+145+150) / 5 = 137

Imputed array:
[137, 137, 137, 137, 137, 120, 130, 140, 145, 150, 137, 137]
         (7 imputed)                (5 from bill)

dados_estimados = true
```

## UNIVERSAL PARSER IMPLEMENTATION

### Class: `UniversalBillParser`

```javascript
// Initialize with distributor profile
const parser = new UniversalBillParser('neoenergia_cosern')

// Parse bill
const result = await parser.parseBill(pdfBuffer)
```

### Key Methods

#### 1. `extractTextFromPDF(pdfBuffer)`
- Uses PDF text layer (primary)
- OCR fallback (if text layer unavailable)
- Returns normalized text

#### 2. `parseClientData(text)` + `parseUnitData(text)`
- Uses regex patterns from ConcessionariaProfile
- Extracts structured data
- Returns null-safe objects

#### 3. `parseConsumptionHistoryWith12Imputation(text)` ✅ **CRITICAL**
- Extracts raw consumption data from bill
- Imputes missing months using available average
- Returns **ALWAYS 12-element array**
- Sets dados_estimados flag
- Prevents dynamic sizing issues

#### 4. `calculateExtractionConfidence()`
- Scores extraction quality (0-100%)
- Penalties for missing required fields
- Additional penalty if dados_estimados = true

## ERROR HANDLING

### Extraction Confidence Penalties

| Field | Penalty | Required |
|-------|---------|----------|
| nome_cliente | -15% | Yes |
| cpf_cnpj_parcial | -10% | Yes |
| endereco | -10% | Yes |
| cidade | -10% | Yes |
| estado | -5% | Yes |
| codigo_cliente | -15% | Yes |
| codigo_instalacao | -20% | Yes ✅ CRITICAL |
| consumo_medio | -15% | Yes |
| dados_estimados flag | -5% | No (bonus deduction) |

### Confidence Interpretation

- **85-100%**: High confidence → AUTO-CREATE project
- **70-85%**: Medium confidence → REVIEW before creation
- **<70%**: Low confidence → MANUAL correction needed
- **Extract fails if**: codigo_instalacao missing (ERR_CRITICAL_UC_CODE_NOT_FOUND)

### Error Codes

| Code | Severity | Category | Meaning |
|------|----------|----------|---------|
| ERR_PROFILE_NOT_FOUND | CRITICAL | DTO_VALIDATION | Distributor not registered |
| ERR_PDF_TEXT_EXTRACTION_FAILED | CRITICAL | FILE_PROCESSING | PDF unreadable |
| ERR_CRITICAL_UC_CODE_NOT_FOUND | CRITICAL | FILE_PROCESSING | UC extraction failed |
| ERR_CONSUMPTION_ARRAY_SIZE_INVALID | CRITICAL | DTO_VALIDATION | Array ≠ 12 elements |
| ERR_BILL_PARSING_FAILED | CRITICAL | FILE_PROCESSING | General parse error |
