📞 SUPPORT & TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Common Issues:

1. "Cannot POST /api/integrations/add-key"
   → Server restart needed (npm run dev in backend)
   → Check that port 3000 is not in use

2. "Rate limiting: Too many attempts"
   → Wait 15 minutes or restart server (resets in-memory state)
   → In production, use Redis for persistent rate limiting

3. "Invalid token" on adding key
   → Login again to refresh JWT token
   → Ensure access token hasn't expired (15 min lifetime)

4. "Key already exists"
   → Each key has unique keyId (timestamp + random)
   → You can add multiple keys of same integration type

5. "Frontend won't load keys"
   → Check browser console (F12) for errors
   → Verify backend is running (http://localhost:3000/api/health)
   → Check that AuthContext is wrapped in main.jsx
